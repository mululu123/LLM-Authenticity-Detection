| 产品 | 地址 | 状态 | 定位 |
|------|------|------|------|

## 战略方向：检测为核心壁垒，内容搞流量

- 技术不是护城河（AI + 开源让门槛降低）
- **6维探针检测**是真正的壁垒：需要持续积累模型指纹库 + 多层检测算法
- 变现路径：检测 API → 中转站认证 → 企业级监控（不急，先搞流量）
- **当前优先级：用自动化内容批量占位长尾词，获取 SEO 流量**

## 内容自动化流水线（待完善）

### 当前状态
- `scripts/generate-articles.js` — 模板拼凑，文章质量低，SEO 价值差
- `scripts/sync-providers.js` — 从 hvoy GitHub 同步供应商数据
- `.github/workflows/sync-providers.yml` — 每天14:00自动同步+生成文章
- n8n 已部署，工作流待设计

### 目标：高质量评测文章批量生产

一篇能拿排名的文章需要的步骤：

1. **素材采集** — 抓取 Reddit、V2EX、知乎、linux.do 等社区对该中转站的真实用户评价
2. **骨架设计** — SEO 友好的标题 + H2/H3 结构 + 长尾词自然布局
3. **内容生产** — 基于真实素材让 LLM 重写改写，不是凭空编造
4. **配图** — 站点截图、账单截图、检测报告截图 + 对比图表 + 封面图
5. **钩子** — 吸引点击但真实的标题和引言（"用了3个月XX站的真实体验"）

### 文章类型规划

| 类型 | 数量潜力 | 示例 |
|------|----------|------|
| 单站评测 | 12篇起 | "XX中转站怎么样？靠谱吗？" |
| 单模型检测报告 | 12×N | "XX站 Claude Opus 真假实测" |
| 价格变动追踪 | 按需 | "5月XX站涨价/降价通知" |
| 新手指南 | 少量 | "API中转站选购避坑指南" |
| 行业深度 | 周更 | "本周中转站市场动态" |

### n8n 工作流设计（待实现）

```
触发（定时/手动）
  → 素材抓取（爬虫/搜索API获取社区评论）
  → LLM 生成初稿（基于真实素材 + Supabase 数据）
  → 配图生成（截图 + AI 封面）
  → 人工审核/编辑（可选）
  → 发布到 Supabase articles 表
  → Vercel 自动展示
```

## 技术架构

```
aiclubin.com 生态
├── llm-test/          ← 主工作区（本目录）
│   ├── src/           ← 探针引擎（Python）
│   ├── scripts/       ← 数据同步、文章生成（JS）
│   ├── visualizer/    ← 前端页面（Vercel 部署）
│   ├── n8n            ← 自动化工作流（已部署）
│   └── Supabase       ← 后端数据库
│
└── new-api-llm/       ← 中转站（独立维护）
    └── new-api/       ← Go 服务，部署后低频维护
```

## 两个项目职责分工

- **llm-test**（本目录）：探针检测、排名、内容生成、SEO、流量 — 主要开发精力在这
- **new-api-llm**：中转站后端优化、新模型适配、稳定性 — 低频维护
